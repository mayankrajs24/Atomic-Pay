CORE ATOMICPAY PRINCIPLE

AtomicPay implements a **three-state payment protocol**:

-1 → Reversed (Tamas)
0 → Transit (Rajas)
+1 → Completed (Sattva)

Rules:

1. All payments must resolve to -1 or +1
2. State 0 must exist only briefly
3. Funds remain locked until commit
4. Any failure triggers automatic rollback
5. No transaction may remain pending

This eliminates suspense accounts and stuck payments.

ATOMIC PAYMENT ENGINE

Extract the atomic transaction logic from the uploaded gateway prototype.

Implement a production version.

Flow:

1 Client sends payment request
2 Gateway creates transaction state = 0
3 Gateway calls sender bank DEBIT
4 Gateway calls receiver bank CREDIT

If both succeed:

state = +1

If any fails:

rollback using UNLOCK
state = -1

Timeout rule:

Transactions exceeding 500ms automatically revert.
