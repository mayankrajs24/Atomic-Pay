Core Concept: Three-State Protocol

Every AtomicPay transaction exists in exactly one of three states. No transaction may remain in state 0.

-1 Tamas (Reversed) ← 0 Rajas (Transit) → +1 Sattva (Completed)

The zero state exists only during the atomic transit window (target: under 500ms). If any failure occurs, the transaction automatically snaps to -1. This eliminates stuck transactions and suspense accounts by design.

Bank Integration Guide

Banks integrate with AtomicPay by implementing a single HTTP endpoint that handles four actions: DEBIT, CREDIT, UNLOCK, and STATUS.

Using the Python SDK

from atomicpay_bank_sdk import AtomicPayBankSDK sdk = AtomicPayBankSDK(bank_name="My Bank", port=6003) sdk.set_accounts({ "ACC_001": {"name": "John", "balance": 10000.0, "active": True} }) sdk.start()

Bank Bridge API

AtomicPay sends POST requests with the following actions:

# DEBIT - Lock funds from sender {"action": "DEBIT", "tx_id": "uuid", "account_id": "ACC_001", "amount": 500} Response: {"state": 1, "reason": "DEBITED", "name": "John", "new_balance": 9500} # CREDIT - Add funds to receiver {"action": "CREDIT", "tx_id": "uuid", "account_id": "ACC_002", "amount": 500} Response: {"state": 1, "reason": "CREDITED", "name": "Jane", "new_balance": 5500} # UNLOCK - Rollback a failed debit {"action": "UNLOCK", "tx_id": "uuid", "account_id": "ACC_001", "amount": 500} Response: {"state": 1, "reason": "UNLOCKED"} # STATUS - Check account {"action": "STATUS", "account_id": "ACC_001"} Response: {"state": 1, "name": "John", "balance": 10000, "active": true}

Payment Flow

1. Client sends POST /api/pay { receiver_mobile, amount, note } 2. Gateway creates transaction (state = 0, Rajas) 3. Gateway calls sender bank: DEBIT 4. Gateway calls receiver bank: CREDIT 5a. Both succeed → state = +1 (Sattva) → COMPLETED 5b. Any fails → UNLOCK sender → state = -1 (Tamas) → REVERSED 6. Transaction signed with HMAC-SHA256 7. Fraud score calculated, AML rules checked 8. Full audit trail recorded

Security

AtomicPay implements the following security measures:

Authentication: JWT tokens with bcrypt PIN hashing Signatures: HMAC-SHA256 on every transaction Fraud: Real-time scoring (velocity, volume, KYC level) AML: Structuring detection, rapid transfer monitoring KYC: 3-level verification (PAN, Aadhaar, Passport) Audit: Complete event trail for regulatory compliance PCI DSS: Tokenized sensitive data, encrypted storage
